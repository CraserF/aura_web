Key Concept

What is a Data Structure?

Data Structure
A specialized format for organizing, processing, retrieving, and storing data. It defines both the logical arrangement and the physical storage in memory.
Key Insight

Choosing the right data structure can make the difference between an algorithm that runs in milliseconds vs. hours.

Start with the formal definition, then use the insight callout to make it tangible. Ask students for real-world examples.

Comparison

Structure Access Insert Delete Search Best For
Array O(1) O(n) O(n) O(n) Index-based access
Linked List O(n) O(1) O(1) O(n) Frequent inserts
Hash Table O(1)* O(1)* O(1)* O(1)* Key-value lookups
Binary Tree O(log n) O(log n) O(log n) O(log n) Sorted data

* Average case -- worst case can be O(n) due to hash collisions

Reveal each row one at a time. Ask students to predict complexity before revealing. Emphasize that there is no single "best" data structure.
Step 4 of 8

Building a Hash Table

  1. Choose a hash function
    Maps keys to array indices uniformly
  2. Allocate the array
    Size should be a prime number for better distribution
  3. Handle collisions
    Use chaining (linked lists) or open addressing
  4. Implement resize
    Grow the table when load factor exceeds 0.75
Walk through each step sequentially. Consider live-coding the hash function step. Pause for questions after collision handling.
Exercise

Knowledge Check

Which data structure would you choose for implementing an undo/redo feature?
A. Array
B. Hash Table
C. Stack
D. Queue
Answer: Stack

Stacks follow LIFO (Last In, First Out) -- perfect for undo operations where you reverse the most recent action first.

Give students 30 seconds to think. Show options one at a time. Ask for hands or use polling. Reveal the answer and explain the reasoning.

Key Takeaways

1

Know Your Options

Different structures excel at different operations -- there is no single "best" structure for every case.

2

Think About Scale

Big-O matters when data grows. O(n²) is fine for 10 items, but disastrous for 10 million.

3

Profile First

Do not optimize prematurely -- measure actual performance before choosing complex structures.

Summarize the three main points. Ask if students have questions. These map back to the learning objectives from slide 2.
